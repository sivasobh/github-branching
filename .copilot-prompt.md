# GitHub Documentation Project - Copilot Instructions

## Project Overview
This documentation project provides comprehensive guidance on GitHub workflows, branching strategies, merge methods, and commit best practices.

## Purpose
Help developers understand:
- Core Git concepts and terminology
- Different branching strategies (Git Flow, GitHub Flow)
- Merging vs Rebasing techniques
- Proper commit practices and conventions
- Release management and hotfix procedures
- Main, develop, and feature branch workflows

## Content Structure
The project is organized into the following sections:
1. **Fundamentals** - Basic Git concepts
2. **Branching** - Branching strategies and workflows
3. **Merging** - Merge vs Rebase explanations
4. **Commits** - Commit best practices
5. **Workflows** - Real-world workflow examples
6. **Release Management** - Release and hotfix procedures
7. **Examples** - Code examples and visual diagrams

## Guidelines for Content
- Use clear, concise explanations
- Include visual ASCII diagrams for complex workflows
- Provide real command examples
- Show both scenarios (recommended and what to avoid)
- Use consistent terminology throughout
- Link between related documents
- Add practical step-by-step guides

## Target Audience
- Developers new to Git/GitHub
- Teams implementing new workflows
- Anyone wanting to understand Git Flow strategy

## Deliverables
- 10 comprehensive documentation files
- ASCII diagrams and visual representations
- Real command examples and workflows
- Ready for GitHub publication
